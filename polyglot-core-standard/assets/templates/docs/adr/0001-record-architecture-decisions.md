# 1. 用 ADR 记录架构决策(尤其是跨语言契约变更)

- 状态:已采纳
- 日期:YYYY-MM-DD

## 背景
在 polyglot-core repo 里,**对 core 跨语言契约的任何改动都是多 sub-tree 改动**:core 一变,
每个 host 的绑定与封装层都要跟着改。这类决策必须在 AI 动手前钉死方向,并留下"为什么否决备选"
以防重走弯路。

## 决策
所有重大决策——特别是 ① core 契约(导出的类型/函数)的增删改、② 新增 host 语言、
③ 绑定姿态(generated vs hand-written)的选择——写成编号、不可变的 ADR(背景 + 选定 +
被否决备选及理由)。AI 只在锁定边界内填实现,且契约变更 + 重新生成的绑定 + 各 host 适配
必须作为**一个**连贯提交落地(绝不拆成绑定过期的中间"绿"态)。

## 后果
跨语言方向可审计;AI 不替我们做含糊取舍,也不重走已排除的路;绑定漂移在 review 层面被提前拦截。
